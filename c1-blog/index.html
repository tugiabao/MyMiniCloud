Giới thiệu dự án MyMiniCloud

🎯 Mục tiêu dự án

MyMiniCloud là một hệ thống hạ tầng Cloud Computing thu nhỏ được xây dựng hoàn toàn bằng Docker Container, mô phỏng cách mà các nhà cung cấp Cloud thực tế (AWS, Google Cloud, Azure) vận hành các dịch vụ.

Tại sao lại là MyMiniCloud?

Trong quá trình học tập môn Cloud Computing, thay vì chỉ sử dụng các dịch vụ Cloud có sẵn, tôi muốn tự tay xây dựng từ đầu một hệ thống hoàn chỉnh để hiểu sâu hơn về cách mà Cloud thực sự hoạt động "bên dưới lớp vỏ".

Ứng dụng thực tế: Smart Aquarium

Để chứng minh hạ tầng MyMiniCloud có thể phục vụ ứng dụng thực tế, tôi đã triển khai hệ thống Smart Aquarium — một bảng điều khiển IoT giám sát và quản lý hồ cá thông minh, bao gồm:

Kiến trúc hệ thống MyMiniCloud

Hệ thống MyMiniCloud gồm 10+ container Docker hoạt động trên một mạng ảo nội bộ (172.19.0.0/16), mỗi container đóng vai trò một "server" độc lập:

Container IP nội bộ Vai trò Công nghệ
c1-sa-web 172.19.0.2 Web Frontend React + Vite + Nginx
c2-sa-api 172.19.0.3 Backend API NestJS + TypeORM
c3 172.19.0.4 Database PostgreSQL 15
c4 172.19.0.5 Authentication Keycloak (OIDC)
c5 172.19.0.6 Object Storage MinIO (S3-compatible)
c6 172.19.0.7 DNS Server BIND9
c7 172.19.0.8 Metrics Collector Prometheus
c8 172.19.0.9 Dashboard Grafana
c9 172.19.0.10 Reverse Proxy / LB Nginx
c10 172.19.0.11 Tunnel (HTTPS) Cloudflare

Load Balancing

Hệ thống sử dụng Nginx Upstream với thuật toán Round-Robin để phân phối request đều giữa 4 instance Backend API:

upstream backend_pool {
    server 172.19.0.3:3000;   # c2-sa-api (gốc)
    server 172.19.0.21:3000;  # c2-api-1
    server 172.19.0.22:3000;  # c2-api-2
    server 172.19.0.23:3000;  # c2-api-3
}

Hướng dẫn triển khai MyMiniCloud

MyMiniCloud đã được triển khai thành công trên 2 nền tảng Cloud:

☁️ AWS EC2

  • Instance: t2.micro (Free Tier)
  • RAM: 1 GB + 4 GB Swap
  • Storage: 26 GB EBS
  • Chế độ: Single instance

🔶 Oracle Cloud

  • Instance: A1.Flex (Free Tier)
  • RAM: 24 GB
  • CPU: 4 vCPU ARM
  • Chế độ: Load Balancing (4 instance)

Bước 1: Clone và cấu hình

# Clone mã nguồn
git clone https://github.com/tugiabao/MyMiniCloud.git
cd MyMiniCloud

# Tạo file biến môi trường
cp .env.example .env
nano .env  # Điền thông tin DB, Auth, Storage...

Bước 2: Khởi động hệ thống

# Khởi động toàn bộ 10+ container
docker compose up -d --build

# Kiểm tra trạng thái
docker ps

Bước 3: Cấu hình Keycloak

# Truy cập: https://auth.azura.io.vn
# 1. Tạo Realm: smart_aquarium
# 2. Tạo Client: aquarium-frontend
# 3. Cấu hình Web Origins: https://sa.azura.io.vn
# 4. Tạo User

Bước 4: Bật Load Balancing (Oracle Cloud)

# Tạo file override cho Load Balancing
cat docker-compose.override.yml

# Rebuild với 4 instance backend
docker compose up -d --build

# Chạy demo
bash demo-lb.sh

💡 Kinh nghiệm rút ra

  • RAM là yếu tố quyết định: AWS Free Tier (1GB) chỉ đủ chạy single instance, cần swap 4GB
  • Docker Compose Override: Cho phép cùng 1 repo chạy trên nhiều môi trường khác nhau
  • Cloudflare Tunnel: Giải pháp HTTPS miễn phí không cần mở port 443
  • ip_hash: Bắt buộc cho WebSocket/Socket.IO khi dùng Load Balancing
